# 运行时错误汇总与修复

## 错误现象

React 组件 `<App>` 连续抛出 `TypeError: Cannot read properties of null (reading 'length')`，导致页面白屏。

## 根本原因

后端 API 在某些场景下返回的字段值为 `null`（例如空会话的 `messages: null`、审批列表的 `approvals: null`），但前端 TypeScript 类型定义为非空数组且代码直接访问 `?.prop.length`，忽略了 `prop` 本身可能为 `null` 的情况。`active?.messages` 在 `active` 非 `null` 但 `messages` 为 `null` 时返回 `null`，再取 `.length` 即崩溃。

## 修复清单

| 文件 | 行号 | 错误代码 | 修复代码 | 触发场景 |
|---|---|---|---|---|
| `App.tsx` | 100 | `[...value.messages].reverse()` | `[...(value.messages ?? [])].reverse()` | 打开会话时 API 返回 `messages: null` |
| `App.tsx` | 242 | `!active?.messages.length` | `!active?.messages?.length` | 渲染控制台时会话无消息且 API 返回 `messages: null` |
| `App.tsx` | 243 | `active?.messages.map(...)` | `active?.messages?.map(...)` | 同上（第 242 行先崩，未实际到达） |
| `App.tsx` | 281 | `approvals.length` | `approvals?.length` | 渲染审批记录页时 API 返回 `approvals: null` |

## 修复模式

对所有可能为 `null` 的数组字段统一采用以下模式之一：

- 安全链：`obj?.field?.length` / `obj?.field?.map(...)`
- 空值合并：`[...(obj.field ?? [])]`

## 建议

后端应确保在无数据时始终返回空数组 `[]` 而非 `null`，从源头消除此类问题。
