## 1. 项目定位

SafeOps Agent 是面向银河麒麟 Kylin Advanced Server OS V11 / LoongArch64 场景的中文 B/S 安全自治运维智能体。项目目标不是提供任意 Shell，而是在自然语言、真实 Linux 证据、MCP 工具、安全策略、人工审批、最小权限执行和可审计 Trace 之间建立受控闭环。

核心产品边界：

- 不提供 `shell.execute`、`terminal.run`、`command.execute` 等任意命令工具。
- MCP Server 与 `safeops-server` 按非 root 运行；受控写动作必须跨 Unix Socket 到 `safeops-privexec`。
- 当前 MCP 工具全部为 L0 只读工具；写动作只存在于本地策略和固定执行器 Handler 中。
- Tool 自报风险不可信；本地 `policies/tools.yaml` 是权威策略来源。
- 不保存模型隐藏 chain-of-thought，只保存结构化决策摘要、证据、Guard/Risk、审批、执行和验证事件。
- LoongArch64 交叉构建不等于目标验证；只有官方 Kylin V11 VM 原生报告可以支持 `TARGET_VERIFIED`。

## 3. 技术栈与入口

后端：

- Go module：`safeops-agent`
- Go 版本目标：`go 1.25.0`
- 默认构建约束：`CGO_ENABLED=0`
- MCP SDK：`github.com/modelcontextprotocol/go-sdk v1.6.1`
- 配置格式：YAML
- 持久化：JSON / JSONL 文件存储

前端：

- React 19 + TypeScript + Vite
- 测试：Vitest + jsdom + Testing Library + axe-core
- API 交互：HTTP JSON + SSE

主要运行入口：

- `cmd/safeops-server`：HTTP API、SSE、静态前端、Agent、Registry 和审批恢复入口。
- `cmd/safeops-privexec`：Unix Socket 固定执行器，默认 dry-run，Lab 模式启用受限 Handler。
- `cmd/mcp-system`、`cmd/mcp-process`、`cmd/mcp-network`、`cmd/mcp-journal`、`cmd/mcp-service`、`cmd/mcp-diagnostic`、`cmd/mcp-file`、`cmd/mcp-config`：8 个 MCP Server。
- `cmd/targetctl`：目标机 probe/test/report/doctor。
- `cmd/safeops-bench`：六套件 Benchmark。
- `web/`：中文控制台。

## 5. MCP 能力扫描

`config/mcp_servers.yaml` 当前声明 8 个启用的内建 MCP Server：

- `system`：系统感知。
- `process`：进程感知。
- `network`：网络感知。
- `journal`：日志感知。
- `service`：服务感知。
- `diagnostic`：关联诊断。
- `file`：allowlist 文件感知。
- `config`：allowlist 配置快照与差异。

`docs/mcp-capability-matrix.md` 和 `policies/tools.yaml` 对应 39 个 L0 只读工具，覆盖 system/process/network/journal/service/diagnostic/file/config 八个域。

当前写能力不是 MCP Tool，策略中只定义以下固定写动作合同：

- `service.restart`
- `service.start`
- `service.stop`
- `process.terminate`
- `file.quarantine`
- `file.restore_quarantine`
- `file.purge_quarantine`

其中永久清理属于 L3，默认拒绝且没有任意命令 Handler。

## 6. API 与前端扫描

后端 API 当前包含：

- 健康检查：`GET /healthz`
- 概览：`GET /api/v1/overview`
- MCP 管理：列出、启用、停用、重新发现、健康检查
- 审批：列出、详情、resolve
- 会话：创建、列表、详情、更新
- 消息：向会话追加用户消息并创建任务
- 任务：列表、详情、Trace、SSE events
- 静态前端：可通过 `-web-root` 服务预构建 Web，并隔离 `/api/` 路由

前端 `web/src/App.tsx` 当前是单页中文控制台，主要页面：

- 智能体控制台
- 系统概览
- 工具中心
- 安全中心
- 根因分析
- 审计追踪

已实现的交互包括会话搜索、归档/恢复、重命名、任务 SSE 跟踪、审批确认、Trace 展示、MCP Server 操作和工作区概览刷新。

## 8. 常用命令

推荐入口：

```bash
npm --prefix web install
make test
make lint
make build-native
make build-linux-amd64
make build-loong64
make benchmark
make release
```

当前主机如果缺少 `make`，可按 `Makefile` 中的底层命令执行。Go 命令在受限环境中应显式使用可写缓存，例如：

```bash
GOCACHE=/tmp/safeops-go-build go test ./...
GOCACHE=/tmp/safeops-go-build go vet ./...
```

开发服务器示例：

```bash
./bin/safeops-server \
  -listen 127.0.0.1:8080 \
  -data ./data \
  -mcp-config ./config/mcp_servers.yaml
```

前端开发服务器：

```bash
npm --prefix web run dev
```

## 9. 主要未完成项与风险

必须继续保持证据边界：

- 官方 Kylin V11 VM 还没有原生安装、运行、targetctl 报告和 Demo 证据。
- LoongArch64 交叉构建只能支持 `IMPLEMENTED` 或 `PARTIAL`，不能支持 `TARGET_VERIFIED`。
- 端口冲突、CPU 高占用、磁盘/日志增长、多轮文件隔离/恢复等 Demo 的后端状态机已测试，但完整 live API/UI/目标机演示仍未完成。
- OpenAI-compatible Provider 缺少真实凭据运行证据；只能声明 contract fake 覆盖。
- `IMPLEMENT.md` 与当前实现阶段不一致，后续应单独修订或降级为历史实现说明。
- 所有新增能力必须继续避免任意命令执行、`sh -c`/`bash -c`、root MCP 和未审批写动作。
